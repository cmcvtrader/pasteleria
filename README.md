# 🧁 Dulces y Postres - Gestión de Obrador y Pedidos

Una aplicación web ligera y progresiva (PWA) diseñada específicamente para la gestión diaria de una repostería u obrador. Permite registrar encargos, llevar un control exacto de los cobros y generar tickets de cocina de forma automatizada.

## ✨ Características Principales

* 📝 **Gestión Ágil de Pedidos:** Formulario rápido para anotar clientes, productos, cantidades y fechas clave (encargo y entrega).
* 💳 **Control de Pagos y Abonos:** Sistema matemático para registrar adelantos parciales (ej. reservas por Bizum) y calcular automáticamente cuánto resta por cobrar en el momento de la entrega.
* 🖨️ **Sistema de Impresión Dual (CSS Print Media):**
  * **Modo Comanda:** Genera un ticket individual con formato de "máquina de escribir", cantidades destacadas y líneas de corte, ideal para pegar en el frigorífico del obrador.
  * **Modo Agenda:** Imprime la lista completa de las entregas pendientes en un formato limpio, ocultando los menús y botones para ahorrar tinta.
* 💰 **Dashboard Financiero Dinámico:** Panel que calcula en tiempo real el total de ventas, el dinero cobrado y la deuda pendiente. Incluye un filtro por rango de fechas para hacer "cierres de caja" semanales o mensuales.
* 🎨 **UI/UX Temática:** Diseño limpio, responsivo y adaptado con una paleta de colores pastel (tonos rosados, aquamarine y azul cielo) que refuerza la identidad visual de una repostería.
* 🔒 **Seguridad y Privacidad:** Protegido con usuario y contraseña mediante autenticación y Row Level Security (RLS) en la base de datos.

## 🛠️ Tecnologías Utilizadas

* **Frontend:** HTML5, CSS3 (Variables, Flexbox, CSS Print Queries), Vanilla JavaScript.
* **Backend / BaaS:** [Supabase](https://supabase.com/) (PostgreSQL, Authentication).
* **Despliegue:** GitHub Pages.

## 🚀 Configuración y Uso

Para desplegar tu propia versión de esta aplicación, sigue estos pasos:

### 1. Base de datos en Supabase
Crea un proyecto en Supabase y ejecuta el siguiente código SQL para crear la tabla necesaria:

```sql
create table public.pasteleria (
  id bigint generated by default as identity not null,
  created_at timestamp with time zone not null default now(),
  cliente text null,
  producto text null,
  precio numeric null,
  fecha_pedido date null,
  fecha_entrega date null,
  estado text null,
  cantidad integer null,
  abono numeric null,
  metodo_pago text null,
  constraint pasteleria_pkey primary key (id)
) TABLESPACE pg_default;
